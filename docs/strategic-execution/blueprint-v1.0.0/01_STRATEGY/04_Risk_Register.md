# Risk Register (ریسک‌ها و کنترل‌ها)

## R1: گیر کردن در پروژه‌های کوچک بی‌کیفیت
- کنترل: فقط Productized Offer + Scope ثابت + Change Request
- Evidence: SOW و Scope signed

## R2: عدم اعتماد بازار دولتی/شرکتی
- کنترل: Proof assets + Reports + Handover Pack
- Evidence: Case Study, Final Report

## R3: SEO بدون Verification
- کنترل: GSC/Bing verification + sitemap submission + monitoring
- Evidence: screenshots + confirmation logs

## R4: وابستگی خارجی در runtime
- کنترل: Dependency Review + Internal fallback plan
- Evidence: Architecture note + runbook
