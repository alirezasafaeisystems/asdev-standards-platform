# اصول اجرای مهندسی‌شده (Execution Principles)

## 1) مرحله‌ای، نه زمان‌بندی
ما deadline محور نیستیم؛ ما **stage-gated** هستیم.
هر Stage فقط وقتی Done است که Evidence و Acceptance Criteria پاس شده باشد.

## 2) Evidence-Driven Delivery
هر خروجی باید قابل نمایش، قابل دفاع، و قابل تحویل باشد.
Evidence می‌تواند:
- لینک production
- اسکرین‌شات
- فایل گزارش
- لاگ تست/مانیتورینگ
- قرارداد و امضا

## 3) استانداردسازی تصمیم‌ها
تصمیم‌های کلیدی باید:
- نوشته شوند
- نسخه بخورند
- در جای درست ذخیره شوند

## 4) کمینه‌سازی جبهه‌ها
در هر چرخه اجرایی فقط روی:
- Portfolio + PersianToolbox + Sales System
تمرکز می‌کنیم. باقی موارد «نگهداری/لانچ حداقلی» هستند.

## 5) کیفیت در نقاط مرگ‌وزندگی
کیفیت جایی مهم است که:
- اعتماد مشتری از بین می‌رود
- پول/پرداخت/داده حساس است
- پایداری و تحویل و گزارش‌پذیری مطرح است
