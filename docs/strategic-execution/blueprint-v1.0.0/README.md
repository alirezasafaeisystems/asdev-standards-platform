# ASDEV Strategic Execution Blueprint (v1.0.0)

این بسته یک «سیستم اجرایی مهندسی‌شده» برای برند، نمونه‌کارها، بازاریابی/فروش، عملیات، و گزارش‌گیری است.  
هدف: جلوگیری از دوباره‌کاری، انحراف از مسیر، و تصمیم‌های مبهم — با فازبندی مرحله‌ای و تسک‌بندی قابل اجرا.

> **اصل کلیدی:** این برنامه **زمان‌بندی ندارد**؛ تنها *مرحله‌ای* و *تسک‌محور* است.  
> هر مرحله فقط زمانی Done می‌شود که **Evidence** و **Acceptance Criteria** آن پاس شود.

---

## 1) این ZIP شامل چیست؟

- اهداف قفل‌شده و تصمیم‌های غیرقابل‌تغییر (Locked Goals)
- نقشه راه برند و نمونه‌کار (Brand & Proof)
- نقشه راه بازاریابی/فروش و قرارداد (Marketing/Sales)
- نقشه راه بلندمدت (Long-term)
- چک‌لیست‌های فازبندی با نقش‌ها (Execution Checklists)
- قالب‌های حرفه‌ای (Proposal / Scope / SOW / Report / Evidence / Meeting Notes)
- سیستم گزارش‌گیری و اتوماسیون (Operations & Automation Pack)

---

## 2) نحوه استفاده (Workflow)

1. از `01_STRATEGY/01_Locked_Goals.md` شروع کن و اهداف را «قفل» نگه دار.
2. یک برد تسک بساز (Trello/Notion/GitHub Projects) با ستون‌های پیشنهادی:
   - Backlog → Ready → In Progress → Evidence Ready → Done
3. برای هر مرحله، فقط تسک‌های همان مرحله را به Ready منتقل کن.
4. هر تسک باید:
   - Owner (نقش مسئول)
   - Definition of Done (DoD)
   - Evidence link / فایل خروجی
   - Acceptance Criteria
5. هفته‌ای یکبار از قالب `04_OPERATIONS/02_Weekly_Review.md` استفاده کن و گزارش بده.

---

## 3) استاندارد نام‌گذاری و ساختار

- **پوشه‌ها** با شماره‌گذاری مرتب شده‌اند تا مسیر مطالعه واضح باشد.
- **فایل‌ها** با پیشوند عددی (`01_`, `02_`...) مرتب می‌شوند.
- اسناد رسمی (قابل ارائه به مشتری) در `05_TEMPLATES/` قرار دارند.
- چک‌لیست‌های اجرایی در `06_CHECKLISTS/` قرار دارند.
