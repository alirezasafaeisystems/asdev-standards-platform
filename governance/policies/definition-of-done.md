# Definition of Done Policy

- Status: Active
- Version: 1.0.0

A change is done when:

1. The requested scope is implemented without unnecessary expansion.
2. Repository quality checks for the changed area pass.
3. Risk-sensitive changes follow human approval gates.
4. Documentation and changelog updates are included when behavior changes.
5. Validation evidence is attached to the PR.
6. No secrets or sensitive data are introduced.
