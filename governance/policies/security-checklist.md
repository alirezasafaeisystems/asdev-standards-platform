# Security Checklist Policy

- Status: Active
- Version: 1.0.0

Review this checklist for every meaningful change:

1. Secrets are not committed.
2. New dependencies are justified and reviewed.
3. Input validation and output encoding are preserved.
4. Authentication/authorization behavior is unchanged unless approved.
5. Sensitive data is not logged.
6. External calls and telemetry are explicit and approved.
7. Security-related docs and runbooks are updated when needed.
